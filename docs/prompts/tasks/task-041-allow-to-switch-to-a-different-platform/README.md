# Task #41 — Allow to switch to a different platform from the output of another

## GitHub Issue

Issue #41: Allow to switch to a different platform from the output of another

## User Request

Let's say I submit an article and select "X" platform.

I complete the share and I want to move on to LinkedIn.

I'd like to "Continue With" from the "X" route to LinkedIn for the same article without typing the URL again.

So each platform route must provide a way to generate the share output for another platform in a single click. That means that the radio inputs should be buttons that generate the output for the new selected platform.

It also means the fetched article content is saved to avoid refetching it.

Only clicking "Start Over" forgets the fetched article.
